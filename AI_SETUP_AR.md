# إعداد مساعد البحث بالذكاء الاصطناعي

تم نشر وظيفة Supabase Edge Function باسم `research-assistant`. لا يعمل توليد المحتوى حتى يُضاف مفتاح OpenAI إلى أسرار المشروع.

## 1. إنشاء مفتاح OpenAI

- افتح لوحة OpenAI API.
- أنشئ مفتاح API جديدًا.
- انسخ المفتاح مرة واحدة واحفظه مؤقتًا في مكان آمن.
- لا ترسله في المحادثات ولا تضعه في GitHub أو ملفات الواجهة.

## 2. إضافة المفتاح إلى Supabase

من مشروع `munjiz`:

1. افتح **Edge Functions**.
2. افتح **Secrets** أو **Manage secrets**.
3. أضف السر التالي:

```text
OPENAI_API_KEY=ضع_المفتاح_هنا
```

يمكن إضافة النموذج اختياريًا:

```text
OPENAI_MODEL=gpt-5-mini
```

4. احفظ التغييرات. لا يلزم إعادة نشر الوظيفة بعد تحديث الأسرار.

## 3. الاختبار

- افتح تطبيق منجِز وسجّل الدخول.
- من الصفحة الرئيسية اختر **مساعد البحث**.
- اكتب موضوعًا واضحًا وحدد النوع والطول والتوثيق.
- اضغط **بحث وكتابة المسودة**.
- راجع المحتوى والمصادر قبل الحفظ أو التسليم.

## الحماية

- وظيفة البحث تتطلب جلسة مستخدم صالحة (`verify_jwt=true`).
- مفتاح OpenAI يُقرأ على الخادم فقط من Supabase Secrets.
- المفتاح لا ينتقل إلى المتصفح ولا يظهر في ملفات البناء.
- الحد الأعلى الحالي للمسودة 2500 كلمة لتقليل التكلفة وزمن التنفيذ.
