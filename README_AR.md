# منجِز — تطبيق PWA لتنظيم تكليفات الطلاب

نسخة عملية موجهة للهاتف، تعمل بطريقتين:

1. **الوضع المحلي:** يعمل مباشرة دون إعداد خادم، ويحفظ البيانات على الجهاز.
2. **الوضع السحابي:** يتفعّل تلقائيًا عند إضافة مفاتيح Supabase، ويضيف التسجيل والمزامنة بين الأجهزة.

## الوظائف المنفذة

- واجهة عربية RTL مناسبة للهاتف.
- إنشاء حساب وتسجيل الدخول بالبريد وكلمة المرور عند تفعيل Supabase.
- استعادة كلمة المرور وتأكيد البريد.
- إضافة التكليفات وتحديد المادة والدكتور والنوع والأولوية والموعد.
- إدارة المهام وحساب نسبة الإنجاز تلقائيًا.
- حذف المهام والتكليفات.
- تصفية التكليفات وعرضها حسب موعد التسليم.
- تعديل الاسم الظاهر في الحساب.
- مؤشر لحالة المزامنة والاتصال.
- تثبيت التطبيق على الشاشة الرئيسية والعمل الأساسي كـ PWA.
- مخطط قاعدة بيانات محمي بسياسات RLS.
- مساحة Storage خاصة ومقيدة بحجم 10 MB للملف، مهيأة للمرحلة التالية.

## التشغيل المحلي

يتطلب Node.js 20 أو أحدث.

```bash
npm install
npm run dev
```

بدون ملف `.env` سيعمل التطبيق في الوضع المحلي مباشرة.

## تفعيل Supabase

1. أنشئ مشروعًا في Supabase.
2. افتح **SQL Editor** ونفّذ الملف:

```text
supabase/schema.sql
```

3. انسخ `.env.example` إلى ملف باسم `.env`.
4. أضف رابط المشروع والمفتاح العام أو المفتاح القابل للنشر:

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_PUBLIC_ANON_KEY
```

5. أعد تشغيل خادم التطوير.
6. من إعدادات Authentication في Supabase، أضف رابط التطبيق إلى Site URL وRedirect URLs.

عند وجود قيم صحيحة في ملف البيئة، تظهر شاشة التسجيل بدل الوضع المحلي.

## الفحص وبناء الإنتاج

```bash
npm run check
npm run build
npm run preview
```

توجد نسخة الإنتاج داخل مجلد `dist`.

## النشر

يمكن رفع المشروع إلى GitHub وربطه بـ Vercel أو Netlify. متغيرا البيئة المطلوبان في منصة النشر هما:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

يجب إضافة رابط النشر النهائي كذلك إلى Redirect URLs في Supabase.

## ما لم يُنفذ بعد

- واجهة رفع الملفات وإدارتها.
- الإشعارات المجدولة الفعلية.
- العمل السحابي دون اتصال مع قائمة انتظار للمزامنة.
- متجر الخدمات أو الدفع.

## مشروع Supabase المرتبط

تم ربط هذه النسخة بمشروع Supabase فعلي باسم `munjiz` في منطقة Mumbai (`ap-south-1`).
لا تُرفع ملفات البيئة إلى GitHub. تُضاف بيانات الاتصال العامة لاحقًا من إعدادات منصة النشر، ولا يحتوي المستودع على أي مفتاح `service_role` أو مفتاح سري.
